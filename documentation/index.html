Virion — Next.js Template

A fintech / SaaS marketing & commerce template, converted from the static HTML template to Next.js 16 (App Router) + React 19 + TypeScript. All 41 pages, faithful to the original design.

Next.js 16 React 19 TypeScript Static export-ready

1. What's inside

This is a faithful port: it reuses the template's original stylesheet (public/assets/css/styles.css + fonts.css) unchanged and reproduces the original markup and class names inside React components, so every page matches the source design. Interactivity (mobile nav, Pages dropdown, pricing/blog tabs, FAQ accordion, cart, marquees, scroll-reveal, demo forms) is handled by a single client component that ports the template's original main.js.

2. Requirements

  • Node.js 20 or newer
  • npm (ships with Node)

3. Getting started

npm install        # install dependencies
npm run dev        # start dev server → http://localhost:3000
npm run build      # production build (all routes prerender static)
npm run start      # serve the production build
npm run typecheck  # TypeScript check

4. Project structure

src/
  app/                 one folder per route; page.tsx generated from source HTML
    layout.tsx         shared shell: links the stylesheet, Header, Footer, scripts
    not-found.tsx      404 boundary
  components/
    layout/            header.tsx, footer.tsx (original markup)
    virion-scripts.tsx client interaction layer (ported from main.js)
scripts/
  convert-all.mjs      HTML → JSX page generator
public/
  assets/css|fonts|images   original template assets (unchanged)
documentation/
  index.html           this file

6. Editing content

Text & images

Page markup lives in the generated src/app/<route>/page.tsx files — edit text and <img> sources directly. Images, fonts and the stylesheet live under public/assets/.

Theme

Brand colors, fonts and the type scale are CSS custom properties in the :root block of public/assets/css/styles.css — change them once to re-theme the whole site.

Header / footer

Shared across all pages — edit once in src/components/layout/header.tsx and footer.tsx.

7. Regenerating pages

The route components are generated from the original HTML by a converter, so a bulk markup change is a re-run rather than 41 hand edits:

node scripts/convert-all.mjs

It extracts each page's content (between the header and footer, which the layout provides), rewrites links to clean routes, normalises asset paths, and converts HTML attributes to JSX.

8. Interactive features

All handled in src/components/virion-scripts.tsx (no jQuery, no Webflow runtime):

  • Mobile navigation & "Pages" mega-dropdown
  • Pricing Monthly/Yearly and blog category tabs
  • FAQ accordion, testimonial & integration marquees
  • Scroll-reveal entrances (respects prefers-reduced-motion)
  • Cart with localStorage persistence, mini-cart, checkout & order confirmation
  • Contact / login / sign-up demo forms (front-end only — wire to your own endpoint)

9. Deployment

The app prerenders every route to static HTML, so it deploys anywhere that runs Next.js.

Vercel (recommended)

npm i -g vercel
vercel            # preview deploy
vercel --prod     # production deploy

Or import the GitHub repository at vercel.com/new — Vercel auto-detects Next.js, no configuration needed.

10. Notes

  • The contact, login, sign-up and checkout forms are front-end demos with no backend. Connect the <form> elements to your own service (Formspree, an API, etc.) to go live.
  • Fonts (Inter, Inter Tight, Island Moments) are self-hosted under public/assets/fonts/ — no external font requests.
